# External Blockers

Per the mission's own instruction: "When blocked by an external dependency: document the exact
blocker, create a fake/replay adapter, continue to the next non-blocked workstream." Every blocker
below was handled that way -- a fake/replay adapter exists and is tested; the real dependency is
what's listed as blocked.

| # | Blocker | Exact dependency missing | Workaround built | Real-mode requirement |
|---|---|---|---|---|
| 1 | XLSX pack import | No spreadsheet-parsing dependency in this repository (`xlsx`/`exceljs`/similar) | CSV/JSON/JSONL/Markdown import fully implemented and tested; `importer.mjs` fails an XLSX pack with a clear `unsupported-format:xlsx`-shaped error rather than a silent no-op | Adding a spreadsheet-parsing npm dependency (out of scope for a no-new-dependency session) |
| 2 | Live PostgresStore | This session's time budget; the JSON backend was prioritized to keep all 15 required commits achievable | 6 SQL migration files, verified to apply cleanly and enforce their constraints against a real Postgres-compatible engine (pglite); a live `PostgresStore` class mirroring `revenue-os/src/store.mjs`'s JSON backend is not implemented | A live `DATABASE_URL` plus building the ~500-line Postgres backend class (the exact pattern already proven in this session's sibling Watchtower mission, just not ported here) |
| 3 | Real payment-provider verification (Stripe/PayPal/Payoneer live API) | Explicitly out of scope by mission design -- workstream 9 asks for evidence-based manual reconciliation, not a live webhook | `payments.mjs` implements the full 13-state machine against fake/replay + manual bank/PayPal/Payoneer *evidence* providers (reference numbers, not API calls) | Real provider API credentials and a live webhook/API integration -- deliberately never built, per "no live payment collection" style constraints repeated across this session's missions |
| 4 | Real website crawling/browser automation | Explicitly disabled by mission design -- "the system must never [check] by itself" in a live-network sense; no headless-browser dependency wired for real use | `providers/crawler.mjs` fake + replay providers back all 18 safe checks; behavior is fully tested against synthetic pages | A real HTTP client or headless browser (Playwright is already a repo dependency, but this package intentionally never calls it), plus a policy for CAPTCHA/anti-bot/robots.txt handling on real target sites |
| 5 | Real outbound email sending (Gmail or any other provider) | Explicitly disabled by mission design -- "the system must never send by itself"; this repository's own `src/gmail.mjs` (real OAuth/send) was deliberately not reused (see `docs/REUSE_VS_REPLACE_DECISION.md`) | `outbound.mjs`'s 5 modes are all local-artifact or fake-replay; `createFakeReplayOutboundProvider` is the only "provider" ever called | Real Gmail OAuth credentials and explicit operator opt-in -- out of scope for this mission entirely |
| 6 | Real statistical significance testing for experiments | Not implemented this session -- no stats dependency, no hand-rolled z-test/chi-square | `funnel.mjs#summarizeExperiment` always reports `significant: false` and only tells the caller whether there is enough sample to consider running a real test | A statistics library (or a carefully hand-verified implementation) computing a real p-value/confidence interval, plus a decision on which test (two-proportion z-test is the obvious default) |
| 7 | Live web server / CSRF protection | No HTTP server was built this session -- workstream 14's UI is a static HTML renderer + a CLI, not a live route | `portal/owner-command-center.mjs` renders to a file; `scripts/cli.mjs` is a local, non-networked CLI | If a live web UI is added later, session/auth/CSRF infrastructure becomes a real, separate build item -- not attempted here |
| 8 | `tests/browser.test.mjs` (pre-existing, this repository) | This environment's Playwright Chromium binary path does not match what `src/browser-crawler.mjs` expects -- confirmed to fail identically on `main` before any of this mission's changes (see TEST_EVIDENCE.md) | None needed from this mission -- this is a pre-existing environmental gap, unrelated to `revenue-os/`, not introduced or worsened here | A working local Playwright browser install matching this repo's pinned version, which this container's setup does not currently provide for that specific pre-existing test |
